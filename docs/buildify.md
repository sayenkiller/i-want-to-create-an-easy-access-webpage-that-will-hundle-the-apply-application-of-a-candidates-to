### Requirements
- Job application system with admin and candidate interfaces
- Admin creates job posts with: title, description, requirements, experience, role, company info, contract type (CDD/CDI), location, remote/onsite, education requirements, number of positions, availability status, post date, company logo
- Candidates apply with: resume upload, cover letter upload, custom fields defined by admin
- Store job posts in 'posts' table
- Store applications in 'participants' table
- Modern, responsive, dynamic UI

### Designs
- Design system with semantic tokens
- Admin job posting page with form
- Candidate application page with dynamic fields
- Job listings view
